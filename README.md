## How to run the code

The main notebook was designed as a full experimental notebook, not as a minimal illustrative example. It executes the workflow end-to-end.

### General execution workflow

1. Open the notebook in **Google Colab**.
2. Install the dependencies from `requirements_v3.txt` if needed.
3. Mount Google Drive if persistent storage is required.
4. Check the configured root paths for:
   - project directory,
   - output directory,
   - and cached data directory.
5. Run the notebook sequentially from the first block to the last block.

### Important execution note

The notebook is organized so that:

- datasets are downloaded or prepared,
- exploratory sweeps are performed,
- DSI-based selection is computed,
- baselines are evaluated,
- final models are trained,
- tables and figures are generated,
- and outputs are saved to persistent folders.

Because of this structure:

- execution may take considerable time,
- some experiments may benefit from GPU availability,
- and several result blocks depend on earlier blocks having completed successfully.

For that reason, the repository also includes pre-generated outputs in `Paper outputs_from_code.zip`, so that the reported materials can be examined without rerunning the entire notebook from scratch.

## Reproducibility notes

The experiments were implemented in Python with a Colab-oriented workflow and rely on common scientific and machine learning libraries, including **PyTorch** and **Scikit-learn**.

The notebook uses fixed seeds and persistent directories in order to improve reproducibility and output organization.

Even so, as in most deep learning experiments, minor run-to-run differences may still occur depending on:

- hardware,
- CUDA availability,
- package versions,
- and execution environment.

The repository therefore provides both:

- the code required to reproduce the pipeline,
- and the generated outputs corresponding to the reported version of the study.
